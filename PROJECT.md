# Stuora QR-Menu — основа проекта

> Единая карта проекта. Читать **этот файл первым** в начале любой сессии — он экономит токены:
> говорит, что где лежит и какой файл трогать под конкретную задачу, чтобы не открывать лишнее.

**Что это.** Статический QR-сайт-меню для бара Stuora (Старая Рига). Гость сканирует QR за столом → в браузере телефона открывается меню. Без приложений, без фреймворков, без сборки.

**Стиль (закон).** Ар-деко × speakeasy: бордовый бархат + золото + кремовый. Промо-сторис ≠ меню: стиль бара взят, но очищен — **один** знак (неон-скрипт «Stuora»), плоская типографика, много воздуха. Полное обоснование — в `identity.md`, решения — в `brief.md`.

## Иерархия файлов — что для чего

| Файл | Роль | Трогаем, когда… |
|---|---|---|
| **`index.html`** | Каркас страницы (шапка, контейнер меню, подвал) | меняем структуру/контакты/подвал |
| **`styles.css`** | Весь дизайн: палитра (CSS-переменные вверху), типографика, карточки, анимации | меняем внешний вид, цвета, отступы |
| **`app.js`** | Логика: `fetch(menu.json)` → рендер вкладок и карточек, scroll-spy, reveal | меняем поведение/рендер, добавляем поля карточки |
| **`menu.json`** | **Контент** (позиции, цены, флаги) — единственный источник данных | **меняем меню, цены, сигнатуры** — только здесь |
| `photos/` | Опц. фото флагманов (`item.image` → `photos/<файл>`) | добавляем реальные фото позже |
| `references/` | Фирменные промо-фиши (язык бренда) | только смотреть, не менять |
| **`brief.md`** | Задание + решения «оставляем/чиним/выбрасываем» | справка по решениям |
| **`identity.md`** | Доказательная база (исследования, кейсы баров) | справка «почему так» |
| **`design-spec.md`** | Что и как реализовано (цвета, шрифты, компоненты, анимации) | справка по реализации |
| `PROJECT.md` | Этот файл — карта и дисциплина | обновляем при смене структуры/этапа |

**Данные ≠ дизайн ≠ каркас.** Правка меню — это `menu.json` и всё. Правка вида — `styles.css`. Позиции **не** захардкожены нигде в html/js.

## Рабочая дисциплина (чтобы не жечь токены)

1. **Сначала — этот файл**, затем открываем только те файлы из таблицы, что нужны для задачи. В остальные не заходим.
2. **Одна задача — свой файл.** Меню → `menu.json`. Цвет/отступ → `styles.css`. Поведение → `app.js`. Не «заодно» править соседнее.
3. **Не переписывать то, что работает.** Правки точечные (Edit), не переписывание файла целиком.
4. **Палитра и шрифты — через переменные** в `:root` (`styles.css`), не хардкодить HEX по месту.
5. **Проверка** после правки: локальный сервер (см. ниже) + взгляд глазами. Не «на веру».

## Как запустить/проверить локально

```
cd "QR StuoraMenu"
python -m http.server 8000
# открыть http://localhost:8000
```
(Нельзя открывать `index.html` двойным кликом как `file://` — `fetch(menu.json)` заблокируется. Только через сервер.)

### ✅ Готово (этап 1 — сайт + интерактив)
- Каркас, дизайн, логика — собраны и проверены локально (все файлы `200`, меню рендерится из JSON, скриншот снят).
- Контент: 4 категории · 30 позиций · 3 сигнатуры · контакты и часы в подвале.
- Неон-лого (текст Great Vibes), sticky-вкладки, scroll-spy, `prefers-reduced-motion`.
- **Интерактив (фото-free, как Dead Rabbit):** каждый напиток — нарисованный бокал (7 типов SVG-линией), который «рисуется» золотом и наливается «цветом вкуса» при прокрутке. **Тап по карточке → раскрытие с составом** (аккордеон).
- Тип бокала и цвет — в `menu.json` (`glass`, `tint`); формы бокалов — объект `GLASS` в `app.js`.

### ✅ Состав вписан (из меню-документа, июль 2026)
- Документ бара сверен: названия/цены/категории совпали 1-в-1 — контент верный.
- Коктейлям добавлен `"ingredients"` (классическая основа + фирменная настойка бара). Настойки/закуски — состав ясен из названия/описания, отдельный список не нужен.
- Состав — для атмосферы/интерактива («из чего этот напиток»), не рецептурник. Пропорции намеренно не указываем. Правка — одна строка в `menu.json`.

## Правила контента `menu.json`

- Позиция: `{ "name", "description", "glass", "tint"?, "ingredients"?: [...], "signature"?: true, "price"?: число }`.
- `glass` — тип бокала: `shot · rocks · highball · martini · coupe · mug · plate` (`plate` = еда, без жидкости). `tint` — «цвет вкуса» (приглушённый, не кислотный).
- `ingredients` — массив строк; показывается в раскрытии карточки. Нет поля → раскрытие показывает описание.
- Цена категории — в объекте категории (`"price": 11`) → в шапке секции «Each 11€».
- Индивидуальная цена (`"price"` у позиции) — только если отличается от цены категории (сейчас так у Kombucha).
- **Сигнатур — максимум 3** (правило menu-engineering: больше акцентов = нет акцентов). Сейчас: Tomato & Sprats Mary, Mushroom Old Fashioned, Smoked Plum Vodka.
